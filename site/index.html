dockwatch

A tiny watchdog for your Docker containers. It watches every container on your machines — local or over SSH — and pushes a notification to your phone the moment one crashes, goes unhealthy, or recovers.

dockwatch — events
01

Three signals. Nothing else.

UNHEALTHY

A container's healthcheck starts failing. You hear about it before your users do.

fires on healthy → unhealthy transition
CRASHED

A container dies with a non-zero exit code. A manual docker stop is recognized and ignored — no false alarms.

fires on running → exited (≠0)
RECOVERED

The container comes back healthy or back up. Close the loop without opening a terminal.

fires on unhealthy/exited → healthy

No noise by design: notifications fire only on state transitions and are rate-limited to one per container per type per 5 minutes — a crash-looping container won't flood your phone. Mute anything with ignore patterns (*-job, backup-*).

Stdlib only. The Docker Engine API is called directly with Go's standard library — no Docker SDK, no client bloat. The single external dependency is golang.org/x/crypto, for SSH.
Survives hiccups. dockwatch subscribes to the Docker events stream and falls back to a 30-second reconcile poll if the stream drops. State lives in a Docker volume; the web UI is embedded in the binary.
03

Remote hosts, zero agents

Add a host from the web UI and dockwatch monitors its Docker daemon through an SSH-forwarded socket. Nothing to install, nothing to update, nothing listening on the remote side.

  • SSH keys & ssh-agent supported (passphrases included)
  • host keys pinned on first use
  • pause / resume hosts with one toggle
  • one dashboard for your whole fleet

04

Running in two minutes

01

Run it

git clone https://github.com/cobanov/dockwatch.git
cd dockwatch
docker compose up -d --build
02

Pick a topic

Open http://localhost:9622, set an ntfy topic, hit Save, then Send test notification. Anyone who knows the topic name can read it — pick something unguessable.

03

Subscribe on your phone

Install the ntfy app (iOS / Android) and subscribe to the same topic. That's it — works with ntfy.sh or your own self-hosted server.
